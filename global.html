          "initial_rpc_timeout_millis": 2000,
          "rpc_timeout_multiplier": 1.5,
          "max_rpc_timeout_millis": 30000,
          "total_timeout_millis": 45000
        }
      },
      "methods": {
        "CreateFoo": {
          "retry_codes_name": "idempotent",

createApiCall(funcWithAuth, settings, optDescriptoropt) → {APICall}

Converts an rpc call into an API call governed by the settings.

In typical usage, func will be a promsie to a callable used to make an rpc request. This will mostly likely be a bound method from a request stub used to make an rpc call. It is not a direct function but a Promise instance, because of its asynchronism (typically, obtaining the auth information).

The result is a function which manages the API call with the given settings and the options on the invocation.

BundleOptions

Properties:
Name Type Description
elementCountThreshold number

the bundled request will be sent once the count of outstanding elements in the repeated field reaches this value.
elementCountLimit number

represents a hard limit on the number of elements in the repeated field of the bundle; if adding a request to a bundle would exceed this value, the bundle is sent and the new request is added to a fresh bundle. It is invalid for a single request to exceed this limit.
requestByteThreshold number

the bundled request will be sent once the count of bytes in the request reaches this value. Note that this value is pessimistically approximated by summing the bytesizes of the elements in the repeated field, and therefore may be an under-approximation.
requestByteLimit number

represents a hard limit on the size of the bundled request; if adding a request to a bundle would exceed this value, the bundle is sent and the new request is added to a fresh bundle. It is invalid for a single request to exceed this limit. Note that this value is pessimistically approximated by summing the bytesizes of the elements in the repeated field, with a buffer applied to correspond to the resulting under-approximation.
delayThreshold number

the bundled request will be sent this amount of time after the first element in the bundle was added to it.

CallOptions

Encapsulates the overridable settings for a particular API call.

CallOptions is an optional arg for all GAX API calls. It is used to configure the settings of a specific API call.

When provided, its values override the GAX service defaults for that particular call.

Typically the API clients will accept this as the second to the last argument. See the examples below.

Type:
  • Object
Properties:
Name Type Attributes Description
timeout number <optional>

The client-side timeout for API calls.
retry RetryOptions <optional>

determines whether and how to retry on transient errors. When set to null, the call will not retry.
autoPaginate boolean <optional>

If set to false and the call is configured for page streaming, page streaming is not performed, instead the callback will be called with the response object.
pageToken Object <optional>

If set and the call is configured for page streaming, page streaming is not performed and requested with this pageToken.
isBundling boolean <optional>

If set to false and the call is configured for bundling, bundling is not performed.
longrunning BackoffSettings <optional>

BackoffSettings used for polling.
promise function <optional>

A constructor for a promise that implements the ES6 specification of promise which will be used to create promises. If not provided, native promises will be used.
Source:
  • gax.js, line 38
Examples
// suppress bundling for bundled method.
api.bundlingMethod(
    param, {optParam: aValue, isBundling: false}, function(err, response) {
  // handle response.
});
// suppress streaming for page-streaming method.
api.pageStreamingMethod(
    param, {optParam: aValue, autoPaginate: false}, function(err, page) {
  // not returning a stream, but callback is called with the paged response.
});
